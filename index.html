Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I'd advise

  • Inform everybody of the proper security practices. For example: Making sure employees don't leave their passwords on a sticky-note taped to their monitor. encourage end-users to use 2FA and a password manager
  • using 2FA/MFA for all logins on the site
  • Make sure the site has TLS (HTTPS)
  • Do not store any password in plaintext, encrypt them using industry standard cryptograhy
  • Hire a pentest firm to check for potential exploits on the website
  • Make sure Network security is up to par, WPA2/3 security protocol on WiFi connectivity.
  • Timing out users for idling too long on the website, requiring them to log in again
  • Sanitization of data. This means making sure the code ran on your website is "authorized" code. This is to prevent XSS attacks
  • Prevent the site from being embedded in an iframe. This is to prevent clickjack attacks
  • Automatic vulnerability scanners automatically scan your website for known threats. This is less thorough than hiring a pentest team and is not a alternative to doing so, but this process is done automatically and regularly.
  • Sanitizing user inputs. This is to prevent malicious users from executing system commands on the host OS
